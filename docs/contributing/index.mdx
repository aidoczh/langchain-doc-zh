# 欢迎贡献者

嗨！感谢您对贡献给 LangChain 的兴趣。作为一个在快速发展的领域中的开源项目，我们非常欢迎各种贡献，无论是新功能、改进基础设施、改善文档还是修复错误。

## 🗺️ 指南

### 👩‍💻 贡献的方式

有很多种方式可以贡献给 LangChain。以下是一些常见的贡献方式：

- [**文档**](/docs/contributing/documentation/style_guide)：帮助改进我们的文档，包括本文档！

- [**代码**](./code.mdx)：帮助我们编写代码，修复错误或改进基础设施。

- [**集成**](integrations.mdx)：帮助我们与您喜欢的供应商和工具进行集成。

- [**讨论**](https://github.com/langchain-ai/langchain/discussions)：帮助回答使用问题并与用户讨论问题。

### 🚩 GitHub 问题

我们的[问题](https://github.com/langchain-ai/langchain/issues)页面会及时更新有关错误、改进和功能请求的内容。

我们有一套标签分类法，以帮助对感兴趣的问题进行分类和发现。请使用这些标签来帮助组织问题。

如果您开始处理一个问题，请将其分配给自己。

如果您要添加一个问题，请尽量将其集中在一个单独的、模块化的错误/改进/功能上。

如果两个问题相关或阻塞，请链接它们而不是合并它们。

我们将尽力保持这些问题的最新状态，尽管在这个领域的快速发展中，有些问题可能会过时。

如果您注意到这种情况，请告诉我们。

### 💭 GitHub 讨论

我们有一个[讨论](https://github.com/langchain-ai/langchain/discussions)页面，用户可以在其中提出使用问题、讨论设计决策并提出新功能。

如果您能够帮助回答问题，请尽量提供帮助！这将使维护者能够更多地专注于开发和错误修复。

### 🙋 获取帮助

我们的目标是拥有尽可能简单的开发者设置。如果您在设置过程中遇到任何困难，请联系维护者！我们不仅希望帮助您解决问题，还希望确保这个过程对未来的贡献者来说是顺利的。

同样地，我们在代码库中执行某些代码规范、格式化和文档标准。

如果您发现这些对您来说很困难（甚至只是烦人）并且影响您的工作，请随时联系维护者寻求帮助 - 我们不希望这些成为阻碍将优秀的代码纳入代码库的障碍。

# 🌟 赞誉

如果您的贡献已经被纳入发布版本，我们将希望在 Twitter 上给予您认可（只有在您愿意的情况下）！

如果您有一个希望我们提及的 Twitter 账号，请在 PR 或通过其他方式告诉我们。
